## 1. 新颖性评分总览

### 1.1 设计目标

新颖性评分是 ThesisMiner v8.0 评估论题创新度的核心机制，从四个维度对候选论题打分，确保生成的论题既有学术价值又有实践意义。设计目标：

1. **多维评估**：从学科交叉、方法迁移、痛点突破、趋势前瞻四个维度综合评估。
2. **量化打分**：每个维度 0-100 分，加权求和得到总分。
3. **阈值过滤**：总分 ≥60 且每维度 ≥40 方为通过。
4. **可解释**：评分细则明确，便于用户理解与改进。

### 1.2 评分公式

```text
总分 = 学科交叉 × 30% + 方法迁移 × 25% + 痛点突破 × 25% + 趋势前瞻 × 20%

通过条件：
  - 总分 ≥ 60
  - 每维度 ≥ 40
```

### 1.3 与 v7 的差异

| 维度 | v7 | v8 |
|------|-----|-----|
| 评分维度 | 可行性 + 创新度 + 谱系贴合度（3 维） | 学科交叉 + 方法迁移 + 痛点突破 + 趋势前瞻（4 维） |
| 评分范围 | 0-10 分 | 0-100 分 |
| 权重 | 0.4 / 0.3 / 0.3 | 0.3 / 0.25 / 0.25 / 0.2 |
| 通过阈值 | ≥6 分 | 总分 ≥60 且每维度 ≥40 |

## 2. 四维创意权重

### 2.1 权重分配

| 维度 | 权重 | 说明 |
|------|------|------|
| 学科交叉 | 30% | 跨学科融合程度 |
| 方法迁移 | 25% | 方法论创新程度 |
| 痛点突破 | 25% | 解决实际问题的程度 |
| 趋势前瞻 | 20% | 与研究趋势的对齐程度 |

### 2.2 权重设计依据

- **学科交叉（30%）**：权重最高，因为跨学科融合是当前学术创新的主要方向，且与导师项目延伸、同门继承策略高度相关。
- **方法迁移（25%）**：方法创新是论题新颖性的重要来源，与跨域联想策略对应。
- **痛点突破（25%）**：解决实际问题是论题价值的核心体现，与矛盾驱动策略对应。
- **趋势前瞻（20%）**：与研究趋势对齐确保论题的时效性，但权重略低以避免过度追逐热点。

## 3. 学科交叉维度（30%）

### 3.1 维度定义

评估论题跨学科融合的程度，包括：

1. **跨学科数量**：论题涉及的学科数量（1-3+）。
2. **学科距离**：涉及学科之间的知识距离（近/中/远）。
3. **融合深度**：跨学科融合的深度（表面引用/方法借鉴/理论融合）。

### 3.2 评分细则

| 分数区间 | 跨学科数量 | 学科距离 | 融合深度 |
|----------|-----------|----------|----------|
| 90-100 | 3+ 学科 | 远距离（如计算机 × 医学 × 心理学） | 理论融合 |
| 80-89 | 2-3 学科 | 中远距离（如计算机 × 医学） | 方法借鉴 |
| 70-79 | 2 学科 | 中等距离（如 NLP × CV） | 方法借鉴 |
| 60-69 | 2 学科 | 近距离（如 NLP × 语音） | 表面引用 |
| 40-59 | 1 学科 | - | 单学科深化 |
| 0-39 | 1 学科 | - | 重复已有工作 |

```text
论题：基于强化学习的医疗问诊安全对齐
涉及学科：计算机科学（强化学习）× 医学（医疗问诊）
学科距离：中远距离
融合深度：方法借鉴

涉及学科：计算机科学（NLP）× 医学（医疗问诊）
```

## 7. 评分细则（0-100）

### 7.1 评分流程

```text
1. 对每个维度独立打分（0-100）
2. 加权求和：总分 = Σ(维度分数 × 权重)
3. 检查通过条件：
   - 总分 ≥ 60
   - 每维度 ≥ 40
4. 输出评分结果 + 评分依据
```

### 7.2 评分实现

```python
NOVELTY_WEIGHTS = {
    "cross_discipline": 0.30,
    "method_transfer": 0.25,
    "pain_point_breakthrough": 0.25,
    "trend_foresight": 0.20
}

PASS_THRESHOLD_TOTAL = 60
PASS_THRESHOLD_DIMENSION = 40

def score_novelty(proposal: dict) -> dict:
    """计算新颖性评分。"""
    scores = {
        "cross_discipline": score_cross_discipline(proposal),
        "method_transfer": score_method_transfer(proposal),
        "pain_point_breakthrough": score_pain_point(proposal),
        "trend_foresight": score_trend_foresight(proposal)
    }

    total = sum(scores[dim] * weight for dim, weight in NOVELTY_WEIGHTS.items())

    passed = (
        total >= PASS_THRESHOLD_TOTAL
        and all(score >= PASS_THRESHOLD_DIMENSION for score in scores.values())
    )

    return {
        "total_score": round(total, 2),
        "dimension_scores": scores,
        "passed": passed,
        "reason": "通过" if passed else "未通过（总分或单维度不达标）"
    }
```

## 8. 相似度计算算法

### 8.1 算法选择

ThesisMiner v8.0 采用**语义嵌入 + 余弦相似度**计算论题与已有研究的相似度：

1. 使用嵌入模型（如 text-embedding-3-small）将论题标题与检索结果标题向量化。
2. 计算余弦相似度。
3. 取最大相似度作为重复度指标。

### 8.2 算法实现

```python
import numpy as np
from openai import OpenAI

def compute_similarity(proposal_title: str, existing_titles: list) -> float:
    """计算论题与已有研究的最大相似度。"""
    client = OpenAI()

    # 获取嵌入向量
    proposal_embedding = get_embedding(client, proposal_title)
    existing_embeddings = [get_embedding(client, t) for t in existing_titles]

    # 计算余弦相似度
    max_sim = 0
    for emb in existing_embeddings:
        sim = cosine_similarity(proposal_embedding, emb)
        max_sim = max(max_sim, sim)

    return max_sim

def cosine_similarity(a: list, b: list) -> float:
    """计算余弦相似度。"""
    a_arr = np.array(a)
    b_arr = np.array(b)
    return np.dot(a_arr, b_arr) / (np.linalg.norm(a_arr) * np.linalg.norm(b_arr))
```

### 8.3 相似度阈值

| 相似度范围 | 风险评级 | 处理方式 |
|-----------|----------|----------|
| < 0.5 | low | 通过 |
| 0.5 - 0.7 | medium | 降权，要求用户确认 |
| ≥ 0.7 | high | 阻断，要求调整差异化方向 |

|-----------|----------|

## 1. 评估准则总览

### 1.1 评估目标与设计哲学

ThesisMiner v8.0 的评估准则体系旨在解决学术论题生成过程中的三大核心问题：

**问题一：新颖性难以量化**  
传统学术评审中，"新颖性"往往依赖评审专家的主观判断，缺乏可复现的量化标准。ThesisMiner 通过将新颖性分解为四个可度量的维度（学科交叉、方法迁移、痛点突破、趋势前瞻），每个维度设定 0-100 分的评分标准，使新颖性评估从主观判断转变为可计算的量化指标。

**问题二：可行性评估维度单一**  
许多论题生成系统仅关注论题的学术价值，忽视了实际研究过程中的资源约束、时间限制和技术门槛。ThesisMiner 的可行性评估覆盖技术可行性、资源可行性、时间可行性和学术可行性四个维度，确保生成的论题不仅在学术上有价值，而且在实际研究中可执行。

**问题三：AI 生成内容难以辨识**  
随着大语言模型的普及，AI 生成的学术文本越来越难以与人类写作区分。ThesisMiner 内置了 200+ 模板词检测列表和置信度评分模型，能够在论题生成阶段就识别并修正 AI 痕迹，确保最终输出的论题符合学术规范。

**设计哲学**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    ThesisMiner 评估设计哲学                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  原则一：量化优先                                                │
│  ─────────────────                                               │
│  所有评估维度均设有 0-100 的量化评分标准，                        │
│  避免模糊的主观判断，确保评估结果可复现、可比较。                  │
│                                                                 │
│  原则二：多维交叉                                                │
│  ─────────────────                                               │
│  单一维度的评估容易产生偏差，ThesisMiner 采用                     │
│  新颖性 × 可行性 × 风格质量 × AI 痕迹四维交叉评估，               │
│  任何单一维度的低分都会影响最终评估结果。                          │
│                                                                 │
│  原则三：阈值驱动                                                │
│  ─────────────────                                               │
│  评估结果通过明确的阈值判定（≥60 通过 / 40-59 待定 / <40 失败），  │
│  而非模糊的"好/中/差"分级，使评估结果可直接用于自动化决策。        │
│                                                                 │
│  原则四：可申诉性                                                │
│  ─────────────────                                               │
│  所有评估结果均支持申诉流程，评估报告包含详细的                    │
│  扣分原因和改进建议，确保被评估者理解评估逻辑并有机会申诉。         │
│                                                                 │
│  原则五：学科适配                                                │
│  ─────────────────                                               │
│  不同学科领域的评估标准存在差异（如理工科侧重技术可行性，          │
│  文科侧重文献基础），评估系统支持按学科领域调整权重和阈值。         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 评估维度权重分配

ThesisMiner v8.0 的综合评分采用加权平均算法，各维度的默认权重如下表所示。权重可根据学科领域和学位级别进行动态调整。

| 评估维度 | 默认权重 | 理学类权重 | 工学类权重 | 文学类权重 | 管理学权重 |
|---------|---------|-----------|-----------|-----------|-----------|
| 新颖性评分 | 30% | 35% | 25% | 30% | 30% |
| 可行性评估 | 30% | 25% | 35% | 25% | 30% |
| 重复度检测 | 15% | 15% | 15% | 20% | 15% |
| 风格质量 | 15% | 15% | 15% | 15% | 15% |
| AI 痕迹 | 10% | 10% | 10% | 10% | 10% |

**学位级别权重微调**：

| 学位级别 | 新颖性微调 | 可行性微调 | 说明 |
|---------|-----------|-----------|------|
| 硕士 | -5% | +5% | 硕士论文更侧重可行性与完成度 |
| 博士 | +5% | -5% | 博士论文更侧重新颖性与创新性 |

**权重配置示例（工学类硕士）**：

```
最终权重 = 基础权重 + 学位微调
新颖性 = 25% + (-5%) = 20%
可行性 = 35% + (+5%) = 40%
重复度 = 15%
风格质量 = 15%
AI 痕迹 = 10%
总计 = 100%
```

### 1.4 评估结果等级定义

| 等级 | 分数范围 | 含义 | 后续动作 |
|------|---------|------|---------|
| A（优秀） | 85-100 | 论题在所有维度均表现优异，可直接进入开题报告撰写 | 自动通过，触发开题报告生成 |
| B（良好） | 70-84 | 论题整体质量良好，个别维度有改进空间 | 通过，附改进建议 |
| C（合格） | 60-69 | 论题达到基本要求，但多个维度需要优化 | 通过，要求修改后重新评估 |
| D（待定） | 40-59 | 论题存在明显缺陷，需要重大修改 | 不通过，提供详细修改建议 |
| E（失败） | 0-39 | 论题在核心维度严重不达标 | 不通过，建议重新选题 |

**一票否决项**：

以下情况无论总分如何，均直接判定为失败（E 级）：

1. **重复度超过 30%**：论题与已有文献的重复度超过 30%，存在学术不端风险
2. **AI 痕迹置信度超过 80%**：论题文本的 AI 生成置信度超过 80%
3. **硬约束违规**：标题格式不合规（超过 20 字、含主动动词等）
4. **时间可行性为零**：研究周期超出学位规定的最长时间限制
5. **学术可行性为零**：文献基线不达标（硕士 <30 篇 / 博士 <50 篇）

## 2. 新颖性评分体系

新颖性评分是 ThesisMiner 评估体系的核心维度，旨在量化论题的创新程度。新颖性评分由四个子维度组成，每个子维度独立评分（0-100 分），最终通过加权平均得到新颖性综合评分。

### 2.1 学科交叉维度（0-100 分）

**定义**：衡量论题涉及的学科领域数量和交叉深度。学科交叉度越高，论题越可能产生创新性的研究成果。

**评分标准**：

| 分数范围 | 等级 | 标准 | 示例 |
|---------|------|------|------|
| 90-100 | 卓越 | 涉及 3 个及以上学科，且有深度的理论融合 | "基于量子计算优化的生物蛋白质折叠预测——计算机科学与生物学的深度融合" |
| 75-89 | 优秀 | 涉及 2 个学科，有明确的理论交叉点 | "基于深度学习的金融风险预警——计算机科学与金融学的交叉应用" |
| 60-74 | 良好 | 涉及 2 个学科，但交叉点较为浅层 | "基于统计学的心理学问卷设计——统计学与心理学的简单结合" |
| 40-59 | 一般 | 涉及 1 个学科的不同分支 | "基于机器学习的图像识别优化——计算机视觉与机器学习的结合" |
| 20-39 | 较弱 | 单一学科单一分支 | "基于 CNN 的图像分类算法改进" |
| 0-19 | 极弱 | 无学科交叉，纯综述性研究 | "深度学习发展综述" |

**学科交叉度计算公式**：

```
学科交叉度 = (学科数量得分 × 0.4 + 交叉深度得分 × 0.4 + 创新潜力得分 × 0.2)

其中：
- 学科数量得分 = min(学科数 × 25, 100)
- 交叉深度得分 = 理论融合度 × 100（由 CriticAgent 评估，0-1 浮点数）
- 创新潜力得分 = 交叉领域研究热度 × 100（基于近 3 年文献增长率）
```

**学科交叉度评估流程**：

```
┌──────────────────────────────────────────────────┐
│  输入：论题标题 + 研究方向 + 研究方法              │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────┐
│  步骤 1：学科识别                                 │
│  ──────────────                                  │
│  使用 NLP 技术从论题中提取学科关键词              │
│  匹配学科分类体系（CS/BIO/PHY/...）              │
│  输出：学科列表 [学科1, 学科2, ...]              │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────┐
│  步骤 2：交叉深度评估                             │
│  ──────────────                                  │
│  分析学科间的理论交叉点                           │
│  评估交叉的深度（浅层应用 / 中层方法 / 深层理论） │
│  输出：交叉深度得分（0-1）                        │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────┐
│  步骤 3：创新潜力评估                             │
│  ──────────────                                  │
│  查询交叉领域的近 3 年文献数量                    │
│  计算文献增长率                                   │
│  输出：创新潜力得分（0-1）                        │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────┐
│  步骤 4：综合评分                                 │
│  ──────────────                                  │
│  学科交叉度 = 学科数量 × 0.4 + 交叉深度 × 0.4    │
│              + 创新潜力 × 0.2                     │
│  输出：学科交叉维度得分（0-100）                  │
└──────────────────────────────────────────────────┘
```

### 2.2 方法迁移维度（0-100 分）

**定义**：衡量论题将某一领域的方法迁移到另一领域的程度。方法迁移是学术创新的重要途径，能够产生"1+1>2"的研究效果。

**评分标准**：

| 分数范围 | 等级 | 标准 | 示例 |
|---------|------|------|------|
| 90-100 | 卓越 | 首次将某方法迁移到全新领域，且有理论支撑 | "将量子退火算法首次应用于社会网络社区发现" |
| 75-89 | 优秀 | 方法迁移有明确的理论依据，且改进了原方法 | "将 Transformer 架构迁移到时间序列预测，并改进注意力机制" |
| 60-74 | 良好 | 方法迁移合理，但改进有限 | "将 CNN 迁移到音频特征提取，调整卷积核维度" |
| 40-59 | 一般 | 方法迁移较为常见，缺乏创新 | "将 SVM 应用于文本分类（已有大量先例）" |
| 20-39 | 较弱 | 方法迁移不恰当或过于简单 | "将线性回归应用于非线性问题" |
| 0-19 | 极弱 | 无方法迁移 | "使用传统方法进行研究" |

**方法迁移度计算公式**：

```
方法迁移度 = (迁移新颖性 × 0.4 + 适配合理度 × 0.3 + 改进贡献度 × 0.3)

其中：
- 迁移新颖性 = 1 - (历史迁移次数 / 100)  # 历史迁移次数越少越新颖
- 适配合理度 = 方法与目标领域的匹配度（0-1，由 CriticAgent 评估）
- 改进贡献度 = 迁移后方法改进的程度（0-1）
```

### 2.3 痛点突破维度（0-100 分）

**定义**：衡量论题解决现有研究痛点的程度。痛点突破度越高，论题的研究价值越大。

**评分标准**：

| 分数范围 | 等级 | 标准 | 示例 |
|---------|------|------|------|
| 90-100 | 卓越 | 解决领域内长期存在的核心痛点 | "解决深度学习模型的可解释性问题——因果推理方法" |
| 75-89 | 优秀 | 解决重要痛点，有明确的技术路线 | "解决联邦学习中的数据异构性问题" |
| 60-74 | 良好 | 解决中等痛点，技术路线可行 | "解决小样本学习中的过拟合问题" |
| 40-59 | 一般 | 解决次要痛点，或痛点不明确 | "优化某算法的运行效率（非核心痛点）" |
| 20-39 | 较弱 | 痛点描述模糊，解决方案不清晰 | "提高某模型的准确率（未说明瓶颈）" |
| 0-19 | 极弱 | 无明确痛点 | "研究某领域的现状" |

**痛点突破度计算公式**：

```
痛点突破度 = (痛点重要性 × 0.4 + 解决方案有效性 × 0.4 + 突破创新性 × 0.2)

其中：
- 痛点重要性 = 痛点在领域内的影响程度（0-1）
- 解决方案有效性 = 方案解决痛点的程度（0-1）
- 突破创新性 = 方案相比已有方法的新颖程度（0-1）
```

### 2.4 趋势前瞻维度（0-100 分）

**定义**：衡量论题对未来研究趋势的把握程度。趋势前瞻度越高，论题越具有长期研究价值。

**评分标准**：

| 分数范围 | 等级 | 标准 | 示例 |
|---------|------|------|------|
| 90-100 | 卓越 | 开创全新研究方向，引领未来趋势 | "基于脑机接口的自适应学习系统——教育技术的未来" |
| 75-89 | 优秀 | 紧跟前沿趋势，有 3-5 年的前瞻性 | "基于大模型的多模态情感计算" |
| 60-74 | 良好 | 符合当前热点，有 1-2 年的前瞻性 | "基于扩散模型的图像生成优化" |
| 40-59 | 一般 | 跟随已有趋势，前瞻性有限 | "基于 BERT 的文本分类改进" |
| 20-39 | 较弱 | 趋势已过时或过于超前 | "基于 RNN 的序列建模（已被 Transformer 取代）" |
| 0-19 | 极弱 | 无趋势意识 | "基于传统统计方法的文本分析" |

**趋势前瞻度计算公式**：

```
趋势前瞻度 = (趋势热度 × 0.3 + 前瞻年限 × 0.3 + 技术成熟度 × 0.2 + 社会需求度 × 0.2)

其中：
- 趋势热度 = 近 1 年相关文献/会议论文的增长率（归一化到 0-1）
- 前瞻年限 = 预计研究方向的热度持续年限（0-1，5 年以上=1.0）
- 技术成熟度 = 技术成熟度曲线位置（0-1，上升期=1.0，衰退期=0.2）
- 社会需求度 = 社会对该研究方向的需求程度（0-1）
```

### 2.5 新颖性综合评分算法

新颖性综合评分通过加权平均四个子维度得出：

```python
def calculate_novelty_score(
    cross_discipline: float,
    method_transfer: float,
    pain_point: float,
    trend_foresight: float,
    weights: dict = None
) -> float:
    """
    计算新颖性综合评分

    Args:
        cross_discipline: 学科交叉维度得分 (0-100)
        method_transfer: 方法迁移维度得分 (0-100)
        pain_point: 痛点突破维度得分 (0-100)
        trend_foresight: 趋势前瞻维度得分 (0-100)
        weights: 各维度权重，默认均等

    Returns:
        新颖性综合评分 (0-100)
    """
    if weights is None:
        weights = {
            "cross_discipline": 0.25,
            "method_transfer": 0.25,
            "pain_point": 0.25,
            "trend_foresight": 0.25,
        }

    # 验证各维度得分范围
    scores = {
        "cross_discipline": max(0, min(100, cross_discipline)),
        "method_transfer": max(0, min(100, method_transfer)),
        "pain_point": max(0, min(100, pain_point)),
        "trend_foresight": max(0, min(100, trend_foresight)),
    }

    # 加权平均
    total_score = sum(scores[k] * weights[k] for k in scores)

    # 惩罚机制：任何单一维度低于 20 分，总分扣减 10 分
    penalty = 0
    for score in scores.values():
        if score < 20:
            penalty += 10

    final_score = max(0, total_score - penalty)

    return round(final_score, 2)
```

```python
def assess_technical_feasibility(
    algorithm_maturity: float,
    data_availability: float,
    compute_requirement: float,
    technical_clarity: float,
    backup_completeness: float
) -> dict:
    """
    评估技术可行性

    Returns:
        {
            "score": float,  # 0-100
            "level": str,    # 卓越/优秀/良好/一般/较弱
            "issues": list,  # 潜在问题列表
            "suggestions": list  # 改进建议
        }
    """
    weights = {
        "algorithm": 0.30,
        "data": 0.25,
        "compute": 0.20,
        "clarity": 0.15,
        "backup": 0.10,
    }

    scores = {
        "algorithm": algorithm_maturity,
        "data": data_availability,
        "compute": compute_requirement,
        "clarity": technical_clarity,
        "backup": backup_completeness,
    }

    total = sum(scores[k] * weights[k] for k in scores)

    issues = []
    suggestions = []

    if scores["algorithm"] < 50:
        issues.append("算法成熟度不足，处于实验或理论阶段")
        suggestions.append("建议选择更成熟的算法，或增加算法验证的预实验")

    if scores["data"] < 50:
        issues.append("数据获取困难，可能影响研究进度")
        suggestions.append("建议提前联系数据提供方，或寻找替代数据集")

    if scores["compute"] < 50:
        issues.append("计算资源需求过高，超出常规设备能力")
        suggestions.append("建议申请云计算资源，或优化算法降低计算需求")

    if scores["clarity"] < 50:
        issues.append("技术路线不清晰，缺乏具体实施步骤")
        suggestions.append("建议细化技术路线图，明确各阶段的技术方案")

    if scores["backup"] < 50:
        issues.append("缺乏备选技术方案，风险较高")
        suggestions.append("建议准备 2-3 个备选技术方案")

    level = (
        "卓越" if total >= 90 else
        "优秀" if total >= 75 else
        "良好" if total >= 60 else
        "一般" if total >= 40 else
        "较弱"
    )

    return {
        "score": round(total, 2),
        "level": level,
        "issues": issues,
        "suggestions": suggestions,
    }
```

def calculate_feasibility_score(
    technical: float,

    temporal: float,

    degree_level: str = "master"
) -> dict:
    """
    计算可行性综合评分

    Args:
        technical: 技术可行性得分 (0-100)

        temporal: 时间可行性得分 (0-100)

            "technical": 0.30,
            "resource": 0.20,
            "temporal": 0.20,
            "academic": 0.30,
        }
    else:
        weights = {
            "technical": 0.25,
            "resource": 0.25,
            "temporal": 0.30,

## 4. 重复度检测算法

重复度检测是防止学术不端的重要手段。ThesisMiner 采用 SimHash、MinHash 和余弦相似度三种算法融合的方式进行重复度检测。

### 4.1 SimHash 算法详解

**算法原理**：SimHash 是一种局部敏感哈希（LSH）算法，能够将相似的文本映射到相似的哈希值。SimHash 的核心思想是将文本特征降维到一个固定长度的二进制指纹，通过计算汉明距离来判断文本相似度。

**算法步骤**：

```
┌──────────────────────────────────────────────────────┐
│                  SimHash 算法流程                     │
├──────────────────────────────────────────────────────┤
│                                                      │
│  输入：文本 T                                         │
│                                                      │
│  步骤 1：分词                                         │
│  ─────────                                           │
│  T → [w1, w2, w3, ..., wn]                           │
│  使用 jieba 分词器进行中文分词                        │
│  去除停用词                                           │
│                                                      │
│  步骤 2：特征权重计算                                 │
│  ─────────────────                                   │
│  对每个词 wi，计算 TF-IDF 权重                        │
│  wi → (wi, weight_i)                                 │
│                                                      │
│  步骤 3：哈希计算                                     │
│  ─────────                                           │
│  对每个词 wi，计算 64 位哈希值                        │
│  wi → hash_i (64-bit binary)                         │
│                                                      │
│  步骤 4：加权累加                                     │
│  ─────────                                           │
│  初始化 V = [0, 0, ..., 0] (64 维)                   │
│  对每个 (wi, weight_i, hash_i):                      │
│      对 hash_i 的第 j 位:                             │
│          if bit == 1: V[j] += weight_i               │
│          else:        V[j] -= weight_i               │
│                                                      │
│  步骤 5：生成指纹                                     │
│  ─────────                                           │
│  对 V 的每一位:                                       │
│      if V[j] > 0: fingerprint[j] = 1                 │
│      else:        fingerprint[j] = 0                 │
│                                                      │
│  输出：64 位 SimHash 指纹                             │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**SimHash 实现代码**：

```python
import hashlib
import jieba
import jieba.analyse
from typing import List, Tuple


class SimHash:
    """SimHash 重复度检测器"""

    def __init__(self, hash_bits: int = 64):
        self.hash_bits = hash_bits

    def _tokenize(self, text: str) -> List[Tuple[str, float]]:
        """分词并计算 TF-IDF 权重"""
        # 使用 jieba 提取关键词及权重
        keywords = jieba.analyse.extract_tags(
            text, topK=20, withWeight=True
        )
        return keywords

    def _hash(self, token: str) -> int:
        """计算 token 的哈希值"""
        hash_obj = hashlib.md5(token.encode("utf-8"))
        hash_hex = hash_obj.hexdigest()
        hash_int = int(hash_hex, 16)
        return hash_int

    def compute(self, text: str) -> int:
        """
        计算文本的 SimHash 指纹

        Args:
            text: 输入文本

        Returns:
            SimHash 指纹 (整数)
        """
        tokens = self._tokenize(text)
        if not tokens:
            return 0

        # 初始化权重向量
        v = [0] * self.hash_bits

        for token, weight in tokens:
            h = self._hash(token)
            for i in range(self.hash_bits):
                bit = (h >> i) & 1
                if bit == 1:
                    v[i] += weight
                else:
                    v[i] -= weight

        # 生成指纹
        fingerprint = 0
        for i in range(self.hash_bits):
            if v[i] > 0:
                fingerprint |= (1 << i)

        return fingerprint

    def hamming_distance(self, fp1: int, fp2: int) -> int:
        """计算两个指纹的汉明距离"""
        x = fp1 ^ fp2
        distance = 0
        while x:
            distance += 1
            x &= x - 1
        return distance

    def similarity(self, fp1: int, fp2: int) -> float:
        """
        计算两个指纹的相似度

        Returns:
            相似度 (0-1)
        """
        distance = self.hamming_distance(fp1, fp2)
        return 1.0 - distance / self.hash_bits
```

### 4.2 MinHash 算法详解

**算法原理**：MinHash 是一种用于估计 Jaccard 相似度的概率算法。通过对文本的 Shingle 集合进行多次哈希，取每次哈希的最小值构成签名，通过签名的相似度来估计原始集合的 Jaccard 相似度。

**算法步骤**：

```
┌──────────────────────────────────────────────────────┐
│                  MinHash 算法流程                     │
├──────────────────────────────────────────────────────┤
│                                                      │
│  输入：文本 T                                         │
│                                                      │
│  步骤 1：Shingling                                   │
│  ─────────                                           │
│  将文本 T 分割为 k-shingle 集合                       │
│  T → {s1, s2, s3, ..., sm}                           │
│  k 通常取 3-5                                        │
│                                                      │
│  步骤 2：生成哈希函数族                               │
│  ─────────────────                                   │
│  生成 n 个独立的哈希函数                              │
│  H = [h1, h2, ..., hn]                               │
│  n 通常取 100-200                                    │
│                                                      │
│  步骤 3：计算 MinHash 签名                            │
│  ─────────                                           │
│  初始化 signature = [∞, ∞, ..., ∞] (n 维)           │
│  对每个 shingle s:                                   │
│      对每个哈希函数 hi:                               │
│          hash_value = hi(s)                          │
│          if hash_value < signature[i]:               │
│              signature[i] = hash_value               │
│                                                      │
│  输出：MinHash 签名 (n 维向量)                        │
│                                                      │
│  相似度计算：                                         │
│  Jaccard ≈ |sig1 ∩ sig2| / n                         │
│  即两个签名相同位置的比例                             │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**MinHash 实现代码**：

```python
import hashlib
import struct
from typing import List, Set, Tuple


class MinHash:
    """MinHash 重复度检测器"""

    def __init__(self, num_perm: int = 128, shingle_size: int = 3):
        self.num_perm = num_perm
        self.shingle_size = shingle_size
        self._init_hash_functions()

    def _init_hash_functions(self):
        """初始化哈希函数族"""
        # 使用 (a*x + b) % p 形式的哈希函数
        self.a = []
        self.b = []
        self.p = (1 << 61) - 1  # 梅森素数
        import random
        random.seed(42)
        for _ in range(self.num_perm):
            self.a.append(random.randint(1, self.p - 1))
            self.b.append(random.randint(0, self.p - 1))

    def _shingle(self, text: str) -> Set[int]:
        """将文本转换为 shingle 集合"""
        shingles = set()
        for i in range(len(text) - self.shingle_size + 1):
            shingle = text[i:i + self.shingle_size]
            h = int(hashlib.md5(shingle.encode()).hexdigest(), 16)
            shingles.add(h)
        return shingles

    def compute(self, text: str) -> List[int]:
        """
        计算文本的 MinHash 签名

        Args:
            text: 输入文本

        Returns:
            MinHash 签名 (n 维列表)
        """
        shingles = self._shingle(text)
        if not shingles:
            return [0] * self.num_perm

        signature = [float('inf')] * self.num_perm
        for s in shingles:
            for i in range(self.num_perm):
                h = (self.a[i] * s + self.b[i]) % self.p
                if h < signature[i]:
                    signature[i] = h

        return [int(s) for s in signature]

    def similarity(self, sig1: List[int], sig2: List[int]) -> float:
        """
        计算两个签名的 Jaccard 相似度估计值

        Returns:
            Jaccard 相似度 (0-1)
        """
        if len(sig1) != len(sig2):
            raise ValueError("签名长度不一致")

        matches = sum(1 for a, b in zip(sig1, sig2) if a == b)
        return matches / len(sig1)
```

### 4.3 余弦相似度算法

**算法原理**：余弦相似度通过计算两个向量之间的夹角余弦值来衡量相似度。在文本相似度计算中，通常使用 TF-IDF 向量。

**实现代码**：

```python
import math
from collections import Counter
from typing import List, Dict


class CosineSimilarity:
    """余弦相似度重复度检测器"""

    def __init__(self):
        self.idf: Dict[str, float] = {}

    def fit(self, documents: List[str]):
        """计算 IDF"""
        N = len(documents)
        df = Counter()
        for doc in documents:
            words = set(doc.split())
            for w in words:
                df[w] += 1

        self.idf = {
            w: math.log(N / (1 + count))
            for w, count in df.items()
        }

    def _tfidf_vector(self, text: str) -> Dict[str, float]:
        """计算文本的 TF-IDF 向量"""
        words = text.split()
        tf = Counter(words)
        total = len(words)

        vector = {}
        for w, count in tf.items():
            tf_val = count / total
            idf_val = self.idf.get(w, 0)
            vector[w] = tf_val * idf_val

        return vector

    def similarity(self, text1: str, text2: str) -> float:
        """
        计算两段文本的余弦相似度

        Returns:
            余弦相似度 (0-1)
        """
        vec1 = self._tfidf_vector(text1)
        vec2 = self._tfidf_vector(text2)

        # 点积
        dot = sum(vec1.get(w, 0) * vec2.get(w, 0) for w in set(vec1) | set(vec2))

        # 模长
        norm1 = math.sqrt(sum(v ** 2 for v in vec1.values()))
        norm2 = math.sqrt(sum(v ** 2 for v in vec2.values()))

        if norm1 == 0 or norm2 == 0:
            return 0.0

        return dot / (norm1 * norm2)
```

### 4.4 多算法融合策略

ThesisMiner 采用三种算法融合的方式进行重复度检测，融合策略如下：

```python
class DuplicationDetector:
    """多算法融合重复度检测器"""

    def __init__(self):
        self.simhash = SimHash()
        self.minhash = MinHash()
        self.cosine = CosineSimilarity()

    def detect(
        self,
        text: str,
        corpus: List[str]
    ) -> dict:
        """
        检测文本与语料库的重复度

        Args:
            text: 待检测文本
            corpus: 语料库文本列表

        Returns:
            {
                "max_duplication": float,  # 最大重复度
                "avg_duplication": float,  # 平均重复度
                "simhash_score": float,    # SimHash 重复度
                "minhash_score": float,    # MinHash 重复度
                "cosine_score": float,     # 余弦相似度
                "fused_score": float,      # 融合重复度
                "matched_sources": list,   # 匹配的来源
                "level": str               # 重复度等级
            }
        """
        # 计算各算法的重复度
        simhash_scores = []
        minhash_scores = []
        cosine_scores = []

        text_fp_simhash = self.simhash.compute(text)
        text_sig_minhash = self.minhash.compute(text)

        for doc in corpus:
            # SimHash
            doc_fp = self.simhash.compute(doc)
            sim = self.simhash.similarity(text_fp_simhash, doc_fp)
            simhash_scores.append(sim)

            # MinHash
            doc_sig = self.minhash.compute(doc)
            sim = self.minhash.similarity(text_sig_minhash, doc_sig)
            minhash_scores.append(sim)

            # Cosine
            sim = self.cosine.similarity(text, doc)
            cosine_scores.append(sim)

        # 取最大值
        max_simhash = max(simhash_scores) if simhash_scores else 0
        max_minhash = max(minhash_scores) if minhash_scores else 0
        max_cosine = max(cosine_scores) if cosine_scores else 0

        # 融合策略：加权平均
        # SimHash 擅长检测整体相似
        # MinHash 擅长检测片段抄袭
        # Cosine 擅长检测主题相似
        fused = (
            max_simhash * 0.35 +
            max_minhash * 0.35 +
            max_cosine * 0.30
        )

        # 确定等级
        if fused >= 0.30:
            level = "严重重复"
        elif fused >= 0.20:
            level = "中度重复"
        elif fused >= 0.10:
            level = "轻度重复"
        else:
            level = "原创"

        return {
            "max_duplication": max(max_simhash, max_minhash, max_cosine),
            "avg_duplication": sum(simhash_scores + minhash_scores + cosine_scores) /
                             (len(simhash_scores) + len(minhash_scores) + len(cosine_scores)),
            "simhash_score": max_simhash,
            "minhash_score": max_minhash,
            "cosine_score": max_cosine,
            "fused_score": fused,
            "level": level,
        }
```

## 5. 风格质量度量

风格质量度量评估论题文本的写作质量，包括句式多样性、词汇丰富度、逻辑连贯性和学术规范性。

### 5.1 句式多样性度量

**定义**：衡量文本中句式结构的丰富程度。

**度量公式**：

```
句式多样性 = 1 - (最长句式占比)

其中：
- 句式类型包括：陈述句、疑问句、感叹句、简单句、复合句、并列句等
- 最长句式占比 = 出现最多的句式类型数量 / 总句数
```

**评分标准**：

| 分数范围 | 句式多样性 | 说明 |
|---------|-----------|------|
| 90-100 | > 0.8 | 句式丰富，多种句式交替使用 |
| 75-89 | 0.6-0.8 | 句式较丰富 |
| 60-74 | 0.4-0.6 | 句式一般，有一定变化 |
| 40-59 | 0.2-0.4 | 句式单一，变化较少 |
| 0-39 | < 0.2 | 句式极度单一 |

### 5.2 词汇丰富度度量

**定义**：衡量文本中词汇的丰富程度。

**度量公式**：

```
词汇丰富度 = TTR (Type-Token Ratio) = 不同词的数量 / 总词数

增强 TTR (MATTR)：
- 使用滑动窗口计算 TTR 的平均值
- 窗口大小通常为 50-100 词
```

**评分标准**：

| 分数范围 | TTR 值 | 说明 |
|---------|--------|------|
| 90-100 | > 0.7 | 词汇非常丰富 |
| 75-89 | 0.5-0.7 | 词汇较丰富 |
| 60-74 | 0.4-0.5 | 词汇一般 |
| 40-59 | 0.3-0.4 | 词汇较贫乏 |
| 0-39 | < 0.3 | 词汇极度贫乏 |

### 5.3 逻辑连贯性度量

**定义**：衡量文本中句子之间的逻辑连贯程度。

**度量方法**：

```
逻辑连贯性 = (连接词密度 × 0.4 + 指代消解率 × 0.3 + 话题连贯度 × 0.3)

其中：
- 连接词密度 = 连接词数量 / 总句数
- 指代消解率 = 成功消解的指代词 / 总指代词
- 话题连贯度 = 相邻句子的话题相关度（0-1）
```

### 5.4 学术规范性度量

**定义**：衡量文本是否符合学术写作规范。

**检查项**：

| 检查项 | 权重 | 说明 |
|--------|------|------|
| 人称规范 | 15% | 是否使用第三人称，避免第一/二人称 |
| 用词规范 | 20% | 是否使用学术用语，避免口语化表达 |
| 引用规范 | 20% | 引用格式是否正确 |
| 数据规范 | 15% | 数据表述是否准确、有来源 |
| 格式规范 | 15% | 标题、段落、列表格式是否规范 |
| 客观性 | 15% | 表述是否客观，避免主观判断 |

```python
def calculate_style_quality(
    sentence_diversity: float,
    vocabulary_richness: float,
    logical_coherence: float,
    academic_normativity: float
) -> dict:
    """
    计算风格质量综合评分

    Returns:
        {
            "score": float,
            "level": str,
            "sub_scores": dict,
            "issues": list,
            "suggestions": list
        }
    """
    weights = {
        "sentence_diversity": 0.20,
        "vocabulary_richness": 0.25,
        "logical_coherence": 0.30,
        "academic_normativity": 0.25,
    }

    scores = {
        "sentence_diversity": sentence_diversity,
        "vocabulary_richness": vocabulary_richness,
        "logical_coherence": logical_coherence,
        "academic_normativity": academic_normativity,
    }

    total = sum(scores[k] * weights[k] for k in scores)

    issues = []
    suggestions = []

    if scores["sentence_diversity"] < 60:
        issues.append("句式多样性不足")
        suggestions.append("建议交替使用不同类型的句式")

    if scores["vocabulary_richness"] < 60:
        issues.append("词汇丰富度不足")
        suggestions.append("建议使用更多学术词汇，避免重复")

    if scores["logical_coherence"] < 60:
        issues.append("逻辑连贯性不足")
        suggestions.append("建议增加连接词，明确逻辑关系")

    if scores["academic_normativity"] < 60:
        issues.append("学术规范性不足")
        suggestions.append("建议检查人称、用词、引用等规范")

    level = (
        "卓越" if total >= 90 else
        "优秀" if total >= 75 else
        "良好" if total >= 60 else
        "一般" if total >= 40 else
        "较弱"
    )

    return {
        "score": round(total, 2),
        "level": level,
        "sub_scores": scores,
        "issues": issues,
        "suggestions": suggestions,
    }
```

### 6.2 检测算法详解

```python
import re
from typing import List, Dict
from collections import Counter


class AITraceDetector:
    """AI 痕迹检测器"""

    # 模板词列表
    TEMPLATE_WORDS = {
        "过渡词": ["综上所述", "总而言之", "由此可见", "值得注意的是", ...],
        "总结词": ["总的来说", "概括而言", "简而言之", ...],
        "强调词": ["值得强调的是", "值得关注的是", ...],
        "学术套话": ["具有重要的理论意义和实践价值", ...],
        "模糊表达": ["在一定程度上", "在某种意义上", ...],
        "连接短语": ["与此同时", "在此基础上", ...],
    }

    def detect(self, text: str) -> dict:
        """
        检测文本中的 AI 痕迹

        Returns:
            {
                "confidence": float,      # AI 生成置信度 (0-1)
                "matched_words": list,    # 匹配的模板词
                "category_scores": dict,  # 各类别得分
                "risk_level": str,        # 风险等级
                "suggestions": list       # 修改建议
            }
        """
        matched = []
        category_counts = {cat: 0 for cat in self.TEMPLATE_WORDS}

        # 扫描模板词
        for category, words in self.TEMPLATE_WORDS.items():
            for word in words:
                count = text.count(word)
                if count > 0:
                    matched.append({
                        "word": word,
                        "category": category,
                        "count": count,
                    })
                    category_counts[category] += count

        # 计算置信度
        total_matches = sum(category_counts.values())
        text_length = len(text)
        density = total_matches / (text_length / 1000)  # 每千字匹配数

        # 置信度模型
        if density > 10:
            confidence = 0.9
        elif density > 7:
            confidence = 0.8
        elif density > 5:
            confidence = 0.7
        elif density > 3:
            confidence = 0.5
        elif density > 1:
            confidence = 0.3
        else:
            confidence = 0.1

        # 风险等级
        if confidence >= 0.8:
            risk_level = "高风险"
        elif confidence >= 0.6:
            risk_level = "中风险"
        elif confidence >= 0.4:
            risk_level = "低风险"
        else:
            risk_level = "安全"

        # 修改建议
        suggestions = []
        if category_counts["过渡词"] > 3:
            suggestions.append("过渡词使用过多，建议减少过渡词的使用频率")
        if category_counts["总结词"] > 2:
            suggestions.append("总结词使用过多，建议精简总结性表述")
        if category_counts["学术套话"] > 2:
            suggestions.append("学术套话过多，建议使用更具体的表述")
        if category_counts["模糊表达"] > 3:
            suggestions.append("模糊表达过多，建议使用更精确的量化表述")

        return {
            "confidence": confidence,
            "matched_words": matched,
            "category_counts": category_counts,
            "risk_level": risk_level,
            "suggestions": suggestions,
        }
```

### 6.3 置信度评分模型

AI 痕迹置信度评分综合以下因素：

```
置信度 = 模板词密度得分 × 0.4
       + 句式模式得分 × 0.3
       + 词汇分布得分 × 0.2
       + 结构特征得分 × 0.1

其中：
- 模板词密度得分 = min(模板词数 / (文本长度 / 1000) / 10, 1.0)
- 句式模式得分 = AI 典型句式匹配率
- 词汇分布得分 = 1 - 词汇分布自然度（AI 文本词汇分布更均匀）
- 结构特征得分 = AI 典型结构特征匹配率
```

### 6.4 误报处理机制

为降低误报率，ThesisMiner 采用以下策略：

1. **白名单机制**：某些学术领域常用表述（如"综上所述"在综述类论文中常见）加入白名单，降低其权重
2. **上下文感知**：结合上下文判断模板词的使用是否合理
3. **学科适配**：不同学科的模板词阈值不同
4. **人工复核**：置信度在 0.5-0.7 之间的结果触发人工复核

### 10.3 常见问题

**Q1：评估结果可以申诉吗？**  
A：可以。所有评估结果均支持申诉，申诉流程详见第 7.3 节。

**Q2：一票否决项有哪些？**  
A：重复度 >30%、AI 痕迹 >80%、硬约束违规、时间可行性为零、学术可行性严重不足。

**Q3：不同学科的评估标准一样吗？**  
A：不一样。评估系统支持按学科领域调整权重和阈值，详见第 9.2 节。

**Q4：硕士和博士的评估标准有什么区别？**  
A：博士论题更注重新颖性（权重 +5%），硕士论题更注重可行性（权重 +5%）。

**Q5：AI 痕迹检测会误报吗？**  
A：会。系统采用白名单机制、上下文感知和学科适配来降低误报率，置信度在 0.5-0.7 之间的结果会触发人工复核。

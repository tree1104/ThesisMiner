## 12. 瓶颈分析

### 12.1 主要瓶颈

1. **LLM 调用**：网络延迟与模型推理时间
2. **数据库 IO**：高并发下的磁盘 IO
3. **内存使用**：大量上下文缓存
4. **前端渲染**：大规模谱系图

### 12.2 瓶颈定位

```python
# 性能分析
import cProfile
import pstats

profiler = cProfile.Profile()
profiler.enable()

# 执行被测代码
run_orchestration()

profiler.disable()
stats = pstats.Stats(profiler)
stats.sort_stats('cumulative')
stats.print_stats(20)
```

## 13. 优化建议

### 13.1 短期优化

1. **连接池优化**：增大数据库连接池
2. **批量操作**：消息批量插入
3. **索引优化**：添加缺失索引
4. **缓存预热**：启动时预加载热点数据

### 13.2 长期优化

1. **读写分离**：SQLite → PostgreSQL
2. **分库分表**：按会话分片
3. **CDN 加速**：前端静态资源
4. **异步化**：更多异步 IO

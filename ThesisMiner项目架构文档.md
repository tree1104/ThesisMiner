基于可实现性与学术合理性，将创意生成技术与真实研究生学术生态深度融合，迭代形成 **ThesisMiner v6.0 – 学术语境驱动的实战论题生成与开题架构**。v6.0 在 v5.0 容忍合理重复与强化创意涌现的基础上，引入“导师对齐机制”、“时间与资源可行性约束”以及“直通开题报告的结构化输出”，确保生成的论题既有创意，又能切实落地完成。

# 项目架构设计文档：ThesisMiner v6.0 – 学术语境驱动的实战论题生成与开题架构

**版本**：6.0  
**核心理念**：生态对齐 · 创意落地 · 直通开题。将 AI 的跨域发散能力与真实的学术生存法则（导师项目、师兄师姐基础、时间与经费限制、开题规范）融合，生成既能通过答辩盲审，又具备可行性的研究论题。  
**架构模式**：启发式多智能体协作 + 学术语境知识图谱 + 创意涌现引擎 + 实战可行性验证器

---

## 0. 与 v5.0 的核心差异

| 维度 | v5.0 | v6.0 |
|------|------|------|
| **语境对齐** | 依赖用户零散知识卡片 | 主动构建“学术谱系图”（导入导师近3-5年论文、同门毕业论文、在研基金项目） |
| **可行性约束** | 依赖模型自评估 | 引入硬性学术日历（硕士生1年内出结果、博士生2年内）与资源边界（经费/设备/数据可得性） |
| **问题驱动** | 矛盾驱动（方法-问题） | 强化“问题意识”，针对人文社科与理工科分别建立痛点挖掘模型 |
| **输出形态** | 生成论题与核心创新点 | 生成论题 + 直接映射开题报告核心模块（背景、意义、国内外现状、研究内容） |
| **规范约束** | 文献引用要求 | 叠加学术格式硬约束（题目限20字、无主动动词；文献基线：硕士≥30篇，博士≥50篇） |

## 1. 全局约束工程（学术实战增强）

| 层级 | 定义 | 处理方式 |
|------|------|----------|
| **学术伦理硬约束** | 抄袭、伪造数据、违背科学规律 | 熔断，拒绝生成并提示风险 |
| **开题规范约束** | 题目超长（>20字）、含主动动词/标点、空泛无关键词 | 自动重写，提取核心变量拼装为规范学术标题 |
| **可行性约束** | 实验周期超期（如硕士超1年）、需昂贵设备但无支撑、数据获取违规 | 降级或否决，Reasoner 需切换至低复杂度方案 |
| **创新软约束** | 与已有工作相似度 0.6-0.85 | 接受，但强制在“差异化声明”中说明技术细节改进 |
| **文献基线约束** | 支撑文献不足 | 标记预警，要求检索扩充，确保国内外研究现状有据可依 |

## 2. 创意涌现引擎（学术生态感知）

在 v5.0 基础上，新增基于真实学术生态的启发式策略，确保论题“有根有基”。

### 2.1 学术谱系链接器
- **导师项目延伸**：解析导师在研项目（国家自然科学基金、社科规划等），将项目子任务转化为论题。生成 Prompt：“基于导师{X项目}的既定研究框架，设计一个能在{1年}内完成的子课题，聚焦于{某个具体参数优化/流程改进}。”
- **同门成果继承**：检索上一届毕业生的论文方向，寻找“未走完的路”。生成 Prompt：“在{师兄论文《Y》}的基础上，将其实验方法迁移至{相邻场景Z}，或引入{新变量W}进行稳健性检验。”

### 2.2 问题意识激发器
- 针对人文社科：扫描社会热点与政策文件，匹配学科理论，寻找现实与理论的张力。
- 针对理工科：从工程应用背景出发，定位“系统故障多”、“算法精度不够”、“耗时过长”等具体痛点作为发起点。

### 2.3 跨域联想与趋势嫁接 (v5.0保留)
- 将领域A的成熟方法嫁接至领域B的未解问题，利用 arXiv 等近期高频术语进行语义组合。

### 2.4 边缘扩展与多候选
- 并行运行上述引擎，保留 3-5 个有差异的方向送入 Reasoner 精炼。

## 3. 直通开题的论题生成与精炼 (Reasoner 3.0)

Reasoner 输出的不再是简单的论题描述，而是直接映射为开题报告的核心素材。

```python
class AcademicThesisProposal(BaseModel):
    title: str                   # 限20字内，无主动动词，包含核心关键词
    inspiration_source: str      # 导师项目/同门继承/跨域联想/矛盾驱动
    problem_awareness: str       # 强烈的问题意识：针对什么现实/理论痛点
    research_significance: dict  # 理论意义(新模型/深化理论) + 实际意义(工程应用/社会贡献)
    literature_review_outline: str # 国内外研究脉络及现有不足
    differentiation: str         # 与最相关工作的具体技术区别
    research_content: list[str]  # 拟解决的关键问题及研究目标
    feasibility_analysis: str    # 时间可行性(符合学制要求)、数据/设备可行性
    confidence_score: float
```

**精炼约束**：
- `title` 必须通过正则校验（长度、词性）。
- `research_significance` 必须双线并行陈述。
- 若方向不可行，返回 `INFEASIBLE` 并建议降低参数维度或更换数据集。

## 4. 学术语境下的新颖性验证器

1. **文献基线检测**：检索并核对相关文献数量，若不足硕士30篇/博士50篇的基线，判定为“研究过窄或伪需求”，要求重新发散。
2. **梯度创新评估（v5.0延续）**：
   - 相似度 <0.4：高创新。
   - 0.4-0.7：常规创新（默认接受）。
   - 0.7-0.85：微创新（需提供明确技术区分）。
   - >0.85：预警，要求重构角度。
3. **自我谱系查重**：不仅对比用户历史，还对比导入的导师与同门论文库，避免课题撞车或完全重复。

---

## 5. 消耗控制与多对话管理

- **按学位分级路由**：硕士论文开题使用中等成本模型；博士论文因需处理超长文献综述（如不少于80篇参考文献，外文不少于30篇），路由至高上下文窗口模型。
- **预算感知**：提供“快速发散（低成本）”与“深度精炼+文献核对（高成本）”两档供用户选择，全程透明计费。

## 7. 迭代开发路径

1. **谱系图谱构建**：开发导入工具，支持解析导师论文库（PDF/知网导出格式）与历年优秀同门毕业论文，构建本课题组的“学术知识图谱”。
2. **开题模板对齐**：依据研究生院标准开题报告模板（选题依据、研究内容、研究方案、工作计划），微调 Reasoner 的 Prompt，确保输出结构直接可用。
3. **硬约束规则编码**：将“题目20字限制”、“硕博文献基线要求”、“1/2年研究周期限制”转化为代码层的硬性拦截规则。
4. **可行性评估闭环**：在生成研究方案后，强制要求模型自证“如何在学习年限内利用现有资源完成”，无法自证则打回重生成。
5. **全流程联调**：打通从“大方向输入”到“谱系检索”、“创意发散”、“可行性过滤”、“开题报告段落生成”的完整链路。
